# Undo/Redo for UI

## Goal
Add Cmd+Z (undo) and Cmd+Shift+Z (redo) to the Hance UI for params + active look changes.

## Scope
**Undoable:** `{ params, activeLook }` snapshots.
**Not undoable:** file upload, look CRUD (create/delete/rename/save-to-disk), look hover previews, timeline scrubbing, panel resizing.

## Design

### History hook
New file: `packages/ui/app/hooks/useHistory.ts`

```ts
type Snapshot = { params: PreviewParams; activeLook: string | null };

function useHistory(initial: Snapshot, cap = 25): {
  state: Snapshot;
  commit: (next: Snapshot) => void;
  undo: () => Snapshot | null;
  redo: () => Snapshot | null;
  canUndo: boolean;
  canRedo: boolean;
}
```

- Two stacks: `past[]`, `future[]`, plus `present`.
- `commit(next)`: push `present` onto `past`, set `present = next`, clear `future`. Trim `past` to cap.
- `undo()`: move `present` to `future`, pop `past` into `present`. Return new present (or null).
- `redo()`: mirror.
- Skip commit if `next` deep-equals `present` (avoid no-op entries).

### Commit-on-release for sliders
Sliders currently call `handleParamChange` on every `onChange`. Change pattern:
- `onChange` — update `params` live (visual feedback), no history commit.
- `onPointerUp` / `onBlur` — call `commit()` with current snapshot.

This requires touching the slider component(s) in `AdjustmentsPanel` to expose a release callback. Discrete inputs (toggles, selects) commit immediately on change.

### Commit points
- Slider release (`pointerup`).
- Toggle/select change.
- `handleLookSelect` (look loaded).
- `handleNoLook` (cleared).
- `handleBatchChange` (if used outside slider flow).

### Keybinding
`useEffect` in `App.tsx`:
- Global `keydown` listener.
- Match `(e.metaKey || e.ctrlKey) && e.key === 'z'`.
- Shift → redo, else undo.
- Skip if `e.target` is an `<input>` / `<textarea>` / `contenteditable` (so text fields in modals keep native undo).
- `preventDefault()` on match.
- On undo/redo, apply returned snapshot to `setParams` and look state.

### Integration with look state
`useLooks` owns `activeLook`. Two options:
1. Lift `activeLook` into `App.tsx` alongside history (simplest).
2. Add a `setActiveLook(name)` setter to `useLooks` that history can call.

Option 2 preserves encapsulation. Needs a look-params loader that doesn't touch history (so undo restoring "Kodachrome" doesn't re-commit).

## Out of scope
- Visible history panel (Looks already serve named-checkpoint role).
- Undoing file uploads or look CRUD.
- Persisting history across page reloads.

## Test plan

- Manual: drag slider, Cmd+Z returns to pre-drag state in one step. Switch look, tweak slider, Cmd+Z twice walks back through both. Cmd+Shift+Z redoes. Focus a text input in New Look modal — Cmd+Z uses native text undo, not app undo.
